# `@repo/ui` — shared React components

A shared library of **React components** for the Aptly monorepo. It is meant to
hold UI primitives that can be reused across front-end apps.

> Part of the [Aptly monorepo](../../README.md).

---

## What's inside

This package currently contains starter components generated by Turborepo:

- `Button`
- `Card`
- `Code`

Each component is exported individually through the package's `exports` map, so
they can be imported directly:

```tsx
import { Button } from "@repo/ui/button";
import { Card } from "@repo/ui/card";
```

---

## Adding a component

A generator is wired up to scaffold new components:

```bash
npm run generate:component --workspace=@repo/ui
```

---

## Scripts

| Script                | What it does                          |
| --------------------- | ------------------------------------- |
| `npm run lint`        | Lint the package (zero warnings).     |
| `npm run check-types` | Type-check with `tsc --noEmit`.       |
| `npm run generate:component` | Scaffold a new React component. |

---

## Note

The Aptly `web` app keeps most of its product-specific UI in its own
`components/` folder. This package is the place for genuinely shared,
app-agnostic primitives as the project grows.
